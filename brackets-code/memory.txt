Brackets Code - Memory & Specification
=============================================

1. WHAT IT IS
-------------
Brackets Code is a CLI code generator for the Brackets (L1VM) assembly language.
It converts natural-language prompts into working .l1com programs.
Written in C, single-file (~8560 lines). Author: Stefan.

2. ARCHITECTURE OVERVIEW
------------------------
The program has a two-tier generation pipeline:

  Prompt -> is_question? -> answer_question() + generate_code()
       |
  smart_generate() --> multi-step split + per-step generation
       | (fallback)
  match_template() --> 31 template functions
       | (fallback)
  gen_hello_world() (default)

Key principle: "planner-first ordering" -- the plan-based smart_generate()
runs BEFORE template matching, so compound/novel prompts are handled
by the planner rather than being captured by shorter template keywords.

3. DATA STRUCTURES
------------------
Program:
  - filename, funcs[] (up to 24), includes[] (up to 24), includes_post[] (up to 24), globals

Function:
  - name, is_local, has_vars, has_vardef, vardef_name
  - vars[] (up to 48 Variable), body (string buffer, up to 64KB)

Variable:
  - name, type, count, values[], num_values

TaskProfile (the plan struct):
  - has_input, input_count, count_value
  - has_literals, literals[], num_literals
  - has_operation, op[] ("add"/"sub"/"mul"/"div"/"mod")
  - has_algorithm, algorithm[], algo_param
  - Enhanced flags: has_sum_range, has_print_even, has_find_max,
    has_fib_seq, has_countdown_from, has_input_sort, has_median,
    has_input_fact, has_string_cat, has_string_compare,
    has_array_assign, has_array_reverse, has_array_find, has_average,
    has_read_file, has_write_file, has_array_vmath,
    has_string_to_num, has_timer,
    has_array_min_max, has_bool_demo, has_bit_check,
    has_max, has_min, has_descending
  - has_loop, loop_start/end/step
  - has_condition, has_output, has_output_str
  - type[] ("int64", "double", "int32", "int16", "byte")
  - title[]

Template struct:
  - keywords (space-separated), description, function pointer

4. GENERATION FLOW (generate_code)
-----------------------------------
1. Check if input is a question -> answer via FAQ
2. Try smart_generate() which:
   a. Splits prompt on conjunctions (multi-step detection)
   b. Validates/merges non-viable steps
   c. Generates per-step code using emitter blocks
   d. Falls through to single-step plan-based generation
3. If plan fails, try match_template() -- scores all 31 templates
4. If template fails, generate default "hello world"

5. PLAN-BASED GENERATOR
-----------------------
5a. Task Parsing (parse_task):
  - Detects type: byte/int8 -> int16/short -> int32 -> int64 -> double/float
  - Extracts numbers (digit + written-out words: "eins"/"one"/...)
  - Count filter: words before "numbers/werte/zahlen/values"
  - Detects operations via ops[] map (add/sub/mul/div/mod in DE+EN)
   - Detects ~45+ patterns (FAQ topics, algorithms, input/output, loops)
   - Enhanced patterns override generic algorithm flags
   - has_string_cat excluded when more specific string flag present
     (reverse, length, vowels, anagram, upper, lower, caesar, palindrome)

5b. Emitter Blocks (NUM_EMITTERS = 76):
  Each emits L1VM code for a specific pattern:

  Block                    Purpose

  emit_math                Pure math with 2-3 literals
  emit_input_loop          Read N numbers, optionally sum
  emit_for_sum             Sum 1..N

  emit_countdown_from       Countdown N to 0
  emit_fib_seq             Fibonacci sequence (first N)
  emit_input_sort           Read N, bubble sort, print (supports descending, max, min)
  emit_median               Read N, sort, print median (uses "two" const 2 for divide)
  emit_string_cat           Input 2 strings, concatenate
  emit_string_compare       Input 2 strings, compare
  emit_array_assign         Input index+value, assign to array
  emit_array_reverse        Read N, reverse array, print
  emit_array_find           Read N + search key, linear search
  emit_input_factorial      Read N, call factorial, print
  emit_array_vmath          Array min/max/average using vmath
  emit_read_file            Read int64 value from file
  emit_write_file           Write int64 value to file
  emit_string_to_num        Parse string input to integer

  emit_pyramid              Print pyramid number pattern 1..n

5c. Multi-Step Composition (split_prompt_steps):
   - Split patterns (ordered longest-first):
     " und dann ", " und danach ", " anschliessend ",
     " and then ", " . ", ". ", "; ", " then ",
     " danach ", " and " (math-guarded), " und " (math-guarded)
   - Math guard: if conjunction surrounded by digits, skip split
   - Each split segment parsed as independent TaskProfile
   - Pre-validation: non-viable steps auto-merge backward
   - Additional merge: "print the median/average/mean/largest/smallest/sum/max/min/them/it/result"
     steps are merged with the preceding step (keeps read+print-the-result as single instruction)
   - Each viable step generates via generate_from_task
   - Variables re-initialized if values differ between steps

5d. generate_from_task dispatch order:
   - Checked in priority order:
     has_input_sort -> emit_input_sort (with descending flag, prints all if !has_max&&!has_min)
     has_median -> emit_median
     has_input_fact, has_string_compare, has_array_* ...
     has_factorial, has_fizzbuzz, has_primes ...
     ...
     has_reverse_string, has_count_vowels, has_anagram_check,
     has_string_to_upper, has_string_to_lower, has_caesar_cipher,
     has_palindrome_string, has_string_length, has_stack, has_queue,
     has_insertion_sort, has_calculator, has_unit_converter,
     has_rock_paper_scissors, has_pyramid ...
     has_string_cat (last among string ops -- less specific than reverse/length/etc)
   - For has_input_sort with has_max: enables dataflow_quiet_mode, emits arr[count-1] only
   - For has_input_sort with has_min: enables dataflow_quiet_mode, emits arr[0] only
   - "print them/it/result" on inherited arrays handled after main dispatch
     (guarded by !has_max && !has_min to avoid double-printing)

7. FAQ SYSTEM
-------------
  Detects questions (starts with was/wie/wer/what/how/why/where/when
  or contains "?"). Answers are hardcoded strings organized by topic.

8. TYPE DETECTION
-----------------
  Order: byte/int8 > int16/short > int32 > int64 > double/float
  Default is int64.

9. vars.l1h INCLUDE
-------------------
  vars.l1h is included only when code uses size constants.
  is_intrinsic_inc() matches both "intr" and "vars" prefixes.

10. INCLUDE ORDERING
--------------------
  - Intrinsic includes (#include <intr-func.l1h>, <vars.l1h>) at top
  - string.l1h, math-lib-vect.l1h, bool.l1h must appear AFTER (funcend)
  - add_include_post() supports deferred includes
  - (funcend) appears before post-function includes

11. CLI INTERFACE
-----------------
  One-shot:    brackets-code "<prompt>" [filename]
  Interactive: brackets-code (no args)
  Commands: /help, /list, /exit, /save <filename>
  Flags: --help/-h, --list/-l, --validate/-v

12. OP MAP (DE+EN support)
--------------------------
  +/plus/add/addiere/sum -> add
  -/minus/subtract/subtrahiere/differenz -> sub
  */mal/multiply/multipliziere/product -> mul
  //geteilt/divide/dividiere/quotient -> div
  %/modulo/mod/rest -> mod

13. KEY DESIGN DECISIONS
-----------------------
  - Planner-first ordering; enhanced patterns clear has_algorithm
  - Multi-step variable conflicts resolved by re-initialization
  - Literal values always use named variables (zero, one, two)
  - vars.l1h added per-emitter rather than globally
  - Complex nested expressions split into single-operation lines
  - input_s requires variable (not literal) for length argument
  - Label newline: L1VM strings do not support escape sequences ("\n")
    -- use separate ":print_s !" and ":print_n !" calls instead
  - string_len calling convention: push string address, call, stpop result
    -- Correct: (str :string_len !) / (len stpop)
    -- Wrong: (str len :string_len !)
  - Const-string declared size must be >= strlen(content) + 1
  - Labels use colon syntax: (:label) for definition, :label for reference
  - "exit" in string constants triggers l1pre macro conflict (exit is a #func in intr-func.l1h)
    -- use "quit" instead of "exit" in menu strings
  - l1pre needs trailing slash on include directory argument

13b. L1VM DOUBLE-PRECISION RULES
---------------------------------
  - Double input: `(val :input_d !)` followed by `(val + zerod val :=)` to coerce
    the value into a proper register (needs `(set const-double 1 zerod 0.0)`)
  - Integer input: `(choice :input_i !)` followed by `(choice + zero choice :=)`
    for similar register coercion
  - Double multiplication: operator BETWEEN operands:
    `(val * c_km_mi result :=)`  -- CORRECT
    `(val c_km_mi * result :=)`  -- WRONG (works for int64 but not double)
  - Print double: `(reset-reg)` BEFORE `(result :print_d !)` to reset double registers
